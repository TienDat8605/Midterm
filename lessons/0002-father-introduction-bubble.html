Unity story system · Lesson 2

Father delivers the opening objective

The implemented prototype waits for the local slime, opens one world-space bubble, types three lines, and closes automatically.

Inspect it

  1. Open Assets/Scenes/NewMap/Map1.unity.
  2. Expand NPCs/FatherNPC/SpeechBubble.
  3. Select Father and inspect FatherIntroduction. The dialogue and four timing values are editable in the Inspector.
  4. Open Assets/Scripts/Story/FatherIntroduction.cs. Find the local-player wait, sequence loop, and maxVisibleCharacters typewriter.

Expected test

Enter the map through the game’s normal single-player or multiplayer flow. After the locally owned slime spawns, wait one second. The three messages should replace each other in the same bubble, then the bubble should disappear.

Why it is local

The component accepts a player without a network view for single-player, a view ID of zero for a local scene/test object, or a Photon view where IsMine is true. No RPC is used for the bubble.

Retrieval check

Which TMP property hides unrevealed characters without changing the complete stored string?

Primary reference: Unity coroutines. Ask your agent if the bubble size, placement, or timing needs adjustment after your first visual test.
