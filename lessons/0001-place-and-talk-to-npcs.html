Unity story system · Lesson 1

Place one NPC and make dialogue appear

The tangible win is one NPC in Map1 that opens a dialogue panel only when the local slime is close enough. Repeat the same pattern for the other two NPCs.

1. Fix the sprite import before placement

  1. Select Assets/Sprites/Story/father.png.
  2. Set Sprite Mode to Single because it contains one character.
  3. Set Pixels Per Unit consistently; 32 is a sensible starting point for these 32–34 px-wide characters.
  4. Open Sprite Editor and use a Bottom Center pivot, then Apply.
  5. Repeat for daughter and guider. Set Filter Mode to Point if the pixel art looks blurry.

The current files are imported as Multiple with a bottom-left custom pivot. That is why dragging and aligning their feet may feel wrong.

2. Place the NPC

  1. Open Assets/Scenes/NewMap/Map1.unity.
  2. Create an empty root named NPCs.
  3. Drag father.png into the scene, rename it FatherNPC, and parent it under NPCs.
  4. Move it to a safe platform. Set Sprite Renderer Sorting Layer/Order so it draws in front of the map.
  5. Add a BoxCollider2D, enable Is Trigger, and resize it around the visible body.

3. Build the light-novel panel

  1. Create Canvas with Screen Space - Overlay and Canvas Scaler set to Scale With Screen Size (1920×1080).
  2. Add a bottom DialoguePanel containing two TextMeshPro UI texts: SpeakerText and BodyText.
  3. Add a small ContinueText saying “Click to continue”. Disable DialoguePanel initially.
  4. Create DialogueUI.cs on the Canvas. It owns the panel and advances through lines.
  5. Create NPCDialogue.cs on each NPC. It owns that NPC’s speaker name and lines.

4. Interaction rule

On a left click, convert Mouse.current.position with Camera.main.ScreenToWorldPoint. Check whether the NPC collider contains that world point, then find a PlayerControllerWithPhysics that has inputEnabled == true. Open dialogue only when the distance is below about 2 world units.

if (Mouse.current.leftButton.wasPressedThisFrame)
{
    Vector2 world = Camera.main.ScreenToWorldPoint(
        Mouse.current.position.ReadValue());

    if (npcCollider.OverlapPoint(world) && LocalSlimeIsNear())
        DialogueUI.Instance.Show(speakerName, lines);
}

This project-specific check matters: remote Photon slimes have input disabled, so they must not trigger another client’s dialogue UI. Dialogue itself should stay local unless synchronized conversations are a deliberate later feature.

5. Inspector data for the three NPCs

  • FatherNPC: Speaker “Father”; lines such as “The mountain remembers every fall.”
  • DaughterNPC: Speaker “Daughter”; assign her own line array.
  • GuiderNPC: Speaker “Guide”; place near the beginning and explain movement/jump controls.

Test loop

  1. Click from far away: nothing opens.
  2. Move the local slime beside the NPC and click its body: panel opens.
  3. Click Continue: exactly one line advances; the last click closes the panel.
  4. While the panel is open, disable the local slime’s inputEnabled; restore its previous value on close.
  5. Run a second Photon client: each player should control only their own dialogue panel.
